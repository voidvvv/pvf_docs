## 一、文件总览

| 文件名 | 作用 | 适用对象 |
|--------|------|----------|
| `itemdropinfo_monster_hell.etc` | **深渊模式**怪物掉率 | 深渊副本（困难 / 非常困难） |
| `itemdropinfo_monseter.etc` | **普通**怪物掉率 | 普通副本中的普通怪物 |
| `itemdropinfo_monseter_extra.etc` | **精英**怪物掉率 | 普通副本中的精英怪 / Named 怪 |
| `worlddrop.etc` | **世界掉落**（按等级段的全局掉落池） | 独立于怪物死亡，1~200 级分段（结构独立，详见第九章） |

> ⚠️ 文件名中 `monseter` 为原版拼写（非 `monster`），修改时**必须保持原名**，否则游戏无法识别。

三个文件共享相同的段落结构，差异在于：
- 深渊文件：用 `[basis of rarity dicision]`（带档位数）+ `[dungeon difficulty drop prob]` 控制稀有度；
- 普通 / 精英文件：用 `[drop prob]`（等级分段掉率）+ 多组 `bonusrate`（倍率）联合控制。

## 二、核心概念（必读）

### 1. 稀有度判定基准 = 百万分之一

所有 `[basis of rarity dicision]` 段中的数值，**以 1 000 000 为基数（=100%）**。
游戏在怪物死亡时roll一个落在 `[0, 1000000)` 区间的随机数，按累计阈值划分稀有度。

### 2. 稀有度顺序（固定）

```
白 → 蓝 → 紫 → 粉 → 史诗
```

### 3. 爆率计算公式（累计阈值差值法）

每一档的**实际爆率 = 本档累计阈值 − 上一档累计阈值**，再除以 1 000 000：

```
某档爆率 = (本档阈值 − 上一档阈值) / 1000000
```

- 第一档（白）的"上一档阈值"视为 **0**；
- 最后一档（史诗）的上限固定为 **1 000 000**（=100%）；
- 每行**最后一个数字**（`1000001` / `1000002`）是**难度/副本引用码**，**不参与稀有度计算**。

## 三、`[basis of rarity dicision]` 稀有度判定段（详解）

### 3.1 段落结构

**深渊文件**（带头部计数值）：
```
[basis of rarity dicision]
    2                                            ← 难度档位数 = 2
    400000  700000  900000  960000  1000000  1000001   ← 困难
    400001  700001  900001  960001  1000001  1000002   ← 非常困难
```

**普通 / 精英文件**（无头部计数，直接 4 行）：
```
[basis of rarity dicision]
    600000  844900  950000  1000000  1000001  1000002   ← 掉落档位 1
    330100  543690  999706  1000000  1000001  1000002   ← 掉落档位 2
    500000  944900  999500  1000000  1000001  1000002   ← 掉落档位 3
    700000  944900  995000  1000000  1000001  1000002   ← 掉落档位 4
```

> 每行 6 列含义：`[白阈值][蓝阈值][紫阈值][粉阈值][史诗上限][难度码]`

### 3.2 计算示例（深渊·困难档）

原始行：`400000  700000  900000  960000  1000000  1000001`

| 稀有度 | 计算 | 爆率 |
|--------|------|------|
| 白 | 400000 − 0 | **40.0%** |
| 蓝 | 700000 − 400000 = 300000 | **30.0%** |
| 紫 | 900000 − 700000 = 200000 | **20.0%** |
| 粉 | 960000 − 900000 = 60000 | **6.0%** |
| 史诗 | 1000000 − 960000 = 40000 | **4.0%** |
| *（末列 1000001 = 困难，不参与计算）* | | |

> 与用户给出的示例（`210000 610000 755000 931000 1000000 1000001`）计算方法一致：粉 = 931000−755000 = 176000 → 17.6%；史诗 = 1000000−931000 = 69000 → 6.9%。

### 3.3 三个文件实际爆率表

#### ① 深渊文件 `itemdropinfo_monster_hell.etc`

**困难（末列 1000001）**：`400000 700000 900000 960000 1000000`

| 白 | 蓝 | 紫 | 粉 | 史诗 |
|----|----|----|----|------|
| 40.0% | 30.0% | 20.0% | 6.0% | 4.0% |

**非常困难（末列 1000002）**：`400001 700001 900001 960001 1000001`

| 白 | 蓝 | 紫 | 粉 | 史诗 |
|----|----|----|----|------|
| 40.0% | 30.0% | 20.0% | 6.0% | 4.0% |

> 💡 **关于 +1 偏移**：非常困难档每个阈值都比困难档大 1。由于"差值"不变，**两档实际爆率完全相同**。+1 仅用于让两套随机数区间不重叠（避免边界判定歧义），**对爆率无实质影响**。

## 四、`[item drop ref table]` 物品等级映射段（详解）

### 4.1 三列含义

每一行三个数字：

```
[地图等级]  [减去的下限]  [加上去的上限]
```

掉落物品的**等级范围**计算：

```
物品等级下限 = 地图等级 − 减去的下限
物品等级上限 = 地图等级 + 加上的上限
```

### 4.2 计算示例

| 原始行 | 地图等级 | 下限计算 | 上限计算 | 物品等级范围 |
|--------|---------|----------|----------|-------------|
| `1  0  15` | 1 | 1−0 = 1 | 1+15 = 16 | **1 ~ 16 级** |
| `5  4  15` | 5 | 5−4 = 1 | 5+15 = 20 | **1 ~ 20 级** |
| `15 10  10` | 15 | 15−10 = 5 | 15+10 = 25 | **5 ~ 25 级** |
| `50  8   3` | 50 | 50−8 = 42 | 50+3 = 53 | **42 ~ 53 级** |
| `110 20  6` | 110 | 110−20 = 90 | 110+6 = 116 | **90 ~ 116 级** |
| `5  4   0`（普通文件低级） | 5 | 5−4 = 1 | 5+0 = 5 | **1 ~ 5 级** |

### 4.3 三个文件的差异规律

| 文件 | 低等级地图（1~12级）物品范围特征 | 高等级地图特征 |
|------|--------------------------------|---------------|
| 深渊 | 范围**宽**（如 `1 0 15` → 1~16，跨度 16） | 后期收窄到 ±3~6 |
| 普通 / 精英 | 范围**窄**（如 `1 0 4` → 1~5，`5 4 0` → 1~5，≈ 地图等级本身） | 80级后稳定在 ±6 |

> 💡 **设计意图**：深渊副本低层就给宽等级范围的装备（爆高等级装备的概率更大）；普通副本前期装备等级紧贴地图等级。

## 五、其他段落说明

### 5.1 `[drop prob]` 等级分段掉率（仅普通 / 精英文件）

#### 解析规则（重要）

`[drop prob count]` 段只有一个数字 **`7`**，它表示 **`[drop prob]` 中每「7 个数字」为一组**（即每组有 7 列）。游戏按这个 count，把 `[drop prob]` 里的数字**每 7 个切一组**依次读取——恰好切出 7 组，对应 7 个等级段。

> ⚠️ **实际文件中，7 组共 7×7=49 个数字全部写在同一行**（Tab 分隔），并非每段一行。下文为便于阅读才分行示意。

**每组 7 个数字的含义**（概率基数 = **1,000,000**，与稀有度判定、世界掉落同一基数）：

| 列 | 字段 | 含义 | 基数 |
|----|------|------|------|
| 1 | 等级下限 | 该段副本 / 怪物等级下界 | — |
| 2 | 等级上限 | 该段副本 / 怪物等级上界 | — |
| 3 | **金币掉落概率** | 该等级段掉落金币的概率 | ÷1,000,000 |
| 4 | （掉落类别 B） | 具体类别待确认（疑似材料 / 消耗品等） | ÷1,000,000 |
| 5 | **装备掉落概率** | 该等级段掉落装备的概率 | ÷1,000,000 |
| 6 | （掉落类别 C） | 具体类别待确认 | ÷1,000,000 |
| 7 | （掉落类别 D） | 具体类别待确认 | ÷1,000,000 |

> 🔑 **关键纠正（基数 = 百万，非 ‰）**：第 3 列（金币）、第 5 列（装备）的概率以 **1,000,000** 为底，**不是千分之（‰）**。例如 `986` = 986 ÷ 1,000,000 = **0.0986%**（若误按 ‰ 则是 98.6%，明显不合理）。第 4、6、7 列的精确类别尚需结合客户端进一步确认，但数值同样是百万分比。

#### 段落结构（普通文件 `itemdropinfo_monseter.etc`，实际为单行）

```
[drop prob count]
    7                              ← 每组 7 个数字（列数）

[drop prob]
    1  15  986  60  309  0   0     ← 第1段：1~15级
    16 23  750  30  230  17  15    ← 第2段：16~23级
    24 27  675  30  209  25  15    ← 第3段
    28 30  675  30  230  17  15    ← 第4段
    31 40  675  20  198  25  10    ← 第5段
    41 50  713  10  162  5   5     ← 第6段
    51 99  750  10  135  8   5     ← 第7段：51~99级（普通文件到此）
```

#### 计算示例（普通文件第 1 段：`1 15 986 60 309 0 0`）

| 列 | 原始值 | 计算（÷1,000,000） | 实际概率 |
|----|--------|--------------------|---------|
| 1~2 | 1 / 15 | — | 等级范围 1~15 级 |
| 3（金币） | 986 | 986 ÷ 1,000,000 | **0.0986%** |
| 5（装备） | 309 | 309 ÷ 1,000,000 | **0.0309%** |

#### 普通 vs 精英对比

| 维度 | 普通文件 | 精英文件 |
|------|---------|---------|
| 第 1 段（1~15 级）原始数据 | `1 15 986 60 309 0 0` | `1 15 986 60 163 15 30` |
| 第 3 列（金币） | 986 | 986（相同） |
| 第 5 列（装备） | 309 → 0.0309% | 163 → 0.0163% |
| 第 6、7 列 | 0 / 0 | 15 / 30（非 0） |
| 等级覆盖范围 | 1~99（末段 `51 99`） | 1~200（末段 `51 200`） |

> 💡 **解读**：
> - 精英文件第 6、7 列出现非 0 值（如 `15 30`），普通文件对应列为 0 → 精英怪多出两类掉落（疑似稀有材料 / 卡片等），体现精英怪掉落更丰富。
> - 精英文件末段延伸到 **200 级**（`51 200 ...`），普通文件止于 **99 级**（`51 99 ...`）。
> - 第 5 列装备概率普通（309）反而高于精英（163），但精英通过第 6、7 列补偿了其他掉落，**不宜单看一列下结论**。

### 5.2 倍率段落（仅普通 / 精英文件）

三组倍率均为**乘法系数**，作用于基础掉率之上：

| 段落 | 含义 | 示例 |
|------|------|------|
| `[party member drop bonusrate]` | **组队人数**倍率（1~4 人） | 4 人组队时部分类别 ×1.6 / ×2.0 |
| `[dungeon difficulty drop bonusrate]` | **副本难度**倍率（普通/冒险/勇士/王者/…） | 王者难度最高 ×2.2 |
| `[monster type drop bonusrate]` | **怪物类型**倍率（普通/精英/Named/Boss） | Boss 最高 ×6.6（普通文件）/ ×4.0（精英文件） |

> 每组倍率按 4~5 个一子组排列，分别对应不同掉落类别。**最终掉率 = 基础掉率 × 组队倍率 × 难度倍率 × 怪物类型倍率**。

### 5.3 深渊文件专属段

| 段落 | 含义 |
|------|------|
| `[dungeon difficulty drop prob]` | 深渊各难度的基础掉落数值（2 组，对应困难 / 非常困难） |
| `[drop prob count]` | = 2，对应 2 个深渊难度 |

### 5.4 普通 / 精英文件尾部段

| 段落 | 含义 |
|------|------|
| `[first boss/named mob hunting]` | 首杀 Boss / Named 怪标记（`1 1` = 启用首杀掉落） |
| `[condition rate]` | 条件掉率（成对数值，如 `90 10` = 90% 概率 + 10% 加成档） |
| `[gold quantity]` | 金币掉落数量档位 |
| `[gold volume]` | 金币掉落数值范围 |

## 六、修改实操指南

### ✅ 需求 1：提高深渊史诗爆率（从 4% → 10%）

修改 `itemdropinfo_monster_hell.etc` 的 `[basis of rarity dicision]`。

**目标**：史诗档从 40000（4%）提升到 100000（10%），需从粉档"借"60000。

| 稀有度 | 原阈值 | 新阈值 | 变化 |
|--------|--------|--------|------|
| 白 | 400000 | 400000 | 不变 |
| 蓝 | 700000 | 700000 | 不变 |
| 紫 | 900000 | 900000 | 不变 |
| 粉 | 960000 | **900000** | 粉降 6%（让出空间） |
| 史诗上限 | 1000000 | 1000000 | 不变 |
| 难度码 | 1000001 | 1000001 | 不变 |

→ 改后：粉 = 900000−900000 = **0%**，史诗 = 1000000−900000 = **10%**。
（实际应从多档均匀分摊，此处仅示意"阈值挪移"原理。）

> 📌 **改爆率的核心操作 = 调整累计阈值的间距**，总和必须保持 1 000 000。

### ✅ 需求 2：让普通副本也能爆史诗

修改 `itemdropinfo_monseter.etc` 的 `[basis of rarity dicision]`，将粉档阈值从 `1000000` 下调，留出史诗空间。例如档位 4：

```
原: 700000  944900  995000  1000000  1000001  1000002
改: 700000  944900  995000  990000   1000000  1000002
```

→ 史诗 = 1000000−990000 = **1%**，粉 = 990000−995000（需同步调整紫/粉使递增合法）。

> ⚠️ **阈值必须严格递增**：`白 < 蓝 < 紫 < 粉 < 史诗上限`，否则判定异常。

### ✅ 需求 3：扩大某等级段掉落装备的等级范围

修改对应文件的 `[item drop ref table]`。例如想让 50 级地图掉 30~70 级装备：

```
原: 50  8  3      （42 ~ 53 级）
改: 50  20 20     （30 ~ 70 级）
```

### ✅ 需求 4：整体提升某类怪物掉率

- 改 `[drop prob]` 后 5 列的 ‰ 数值（直接调基础掉率）；
- 或改 `[monster type drop bonusrate]` / `[dungeon difficulty drop bonusrate]` 的倍率（间接放大）。

---

## 七、修改注意事项

1. **保持文件名不变**（含 `monseter` 原始拼写）。
2. **基数固定 1 000 000**：所有 `[basis of rarity dicision]` 累计阈值最后一档必须 = 1 000 000。
3. **阈值严格递增**：白 < 蓝 < 紫 < 粉 < 史诗。
4. **末列难度码勿改**：`1000001` / `1000002` 是副本 / 难度引用，改动会导致掉落表错位。
5. **tab 分隔**：段落内数值用 Tab 分隔，段落名用 `[]` 包裹，修改时保留原有空白格式。
6. **改完需重新打包 PVF**：编辑 `.etc` 文本后，必须用 pvfUtility 等工具重新导入并另存为 PVF 封包，游戏方能生效。
7. **等级范围合理性**：`[item drop ref table]` 中"减去的下限"不要超过地图等级（否则下限为负），"加上去的上限"过大可能导致掉落不存在的物品等级。

---

## 八、附：爆率速算公式卡

```
【稀有度爆率】
某档爆率 = (本档累计阈值 − 上一档累计阈值) / 1000000 × 100%

【物品等级范围】（[item drop ref table]）
下限 = 地图等级 − 第2列
上限 = 地图等级 + 第3列

【drop prob 等级分段掉率】（普通/精英文件 [drop prob]）
[count] = 每组数字个数（普通/精英均为 7 = 7 列）；每组前 2 列为等级范围，第 3 列 = 金币掉率、第 5 列 = 装备掉率，基数均为 1,000,000（986 → 0.0986%，309 → 0.0309%）；第 4/6/7 列类别待确认。

【最终掉率】（普通/精英文件）
最终 = 基础掉率 × 组队倍率 × 难度倍率 × 怪物类型倍率

【世界掉落】（worlddrop.etc）
每个物品独立判定，实际爆率 = 物品对第 2 列数值 ÷ 1,000,000（1000 = 千分之一 = 0.1%，1,000,000 = 必掉，0 = 不掉）
```

---

## 九、`worlddrop.etc` 世界掉落

> 文件路径：`huanjing/etc/worlddrop.etc`
> 作用：定义**世界掉落**物品池——独立于怪物掉率表之外，按**地图 / 怪物等级**分段掉落的固定物品清单（药水、材料、卡片、副本基础奖励等）。一只怪死亡时，除了走怪物掉率表，还会**独立判定**一次世界掉落池。

### 9.1 文件总体结构

整段被 `[world drop]` ... `[/world drop]` 包裹，内部由**若干"等级块"顺序排列**，等级块之间用**单独一行**的 **`-1`** 分隔。

```
[world drop]
    <等级块 1>
    -1
    <等级块 2>
    -1
    ...
    <等级块 200>
    -1
[/world drop]
```

本文件共 **200 个等级块**，对应等级 **1 ~ 200**（与怪物掉率文件 `[item drop ref table]` 的地图等级上限一致）。文件末尾结构：第 23103 行 `200 0`（最后一块），第 23218 行 `-1`，第 23219 行 `[/world drop]`。

### 9.2 单个等级块结构

每个等级块由 **1 行头部 + 若干行物品对**组成：

```
<等级>   0              ← 头部：等级号 + 固定 0（保留位，本文件恒为 0）
<物品ID> <掉率>          ← 物品对，两个一组
<物品ID> <掉率>
...
```

**字段含义：**

| 位置 | 字段 | 含义 | 说明 |
|------|------|------|------|
| 头部第 1 列 | **等级** | 该块对应的地图 / 怪物等级 | 从 1 递增到 200 |
| 头部第 2 列 | 保留位 | 固定占位 | **本文件恒为 0**，修改时保持 0 |
| 物品行第 1 列 | **物品 ID** | 掉落物品编号 | 对应 equipment / stackable 等 LST 表中的物品 |
| 物品行第 2 列 | **掉落概率** | 百万分比，实际爆率 = 数值 ÷ 1,000,000 | `1000` = 0.1%（千分之一），`1,000,000` = 100% 必掉，`0` = 不掉落 |

### 9.3 概率取值规律（实测归纳）

> ⚠️ **概率基数 = 1,000,000**：worlddrop 的掉率与 `[basis of rarity dicision]` 使用**同一基数**，实际爆率 = **数值 ÷ 1,000,000**。即 `1000` = 千分之一 = 0.1%（**并非必掉**）；要 100% 必掉需填 `1,000,000`。

| 掉率值 | 计算（÷ 1,000,000） | 实际爆率 | 文件中典型物品 |
|--------|---------------------|---------|---------------|
| `0` | 0 | **0%（不掉落）** | 大量 `2500xxx`（装扮券）、低等级段的 `2600xxx` |
| `1` | 1 / 1,000,000 | 0.0001%（百万分之一） | `2610001~2610005`（卡片）低等级段 |
| `2` | 2 / 1,000,000 | 0.0002% | `2610006 / 2610007` |
| `10` | 10 / 1,000,000 | 0.001%（十万分之一） | `2600xxx` 高等级段（200 级块大量为 10） |
| `50` | 50 / 1,000,000 | 0.005% | `1047`（几乎每块结尾出现） |
| `150` | 150 / 1,000,000 | 0.015% | `7454` |
| `500` | 500 / 1,000,000 | 0.05%（万分之五） | `8669`、`690000280` |
| `600` | 600 / 1,000,000 | 0.06% | `1111`、`1117`（200 级块） |
| `1000` | 1000 / 1,000,000 | 0.1%（千分之一） | `7248`、按等级激活的 `1106 / 1107 / 1108` |

> 💡 **三条关键规律：**
> 1. **同一物品在不同等级块可有不同掉率**。典型是 `1106 / 1107 / 1108` 这组随等级升级的系列物品：
>    - 1~5 级块：`1106` 激活（0.1%）、`1107 / 1108` 为 0；
>    - 6~15 级块：`1107` 激活（`1106` 退场）；
>    - 16 级起：`1108` 激活。
> 2. **`8669 / 7454 / 690000280 / 7248`** 几乎在所有等级块中以相同掉率出现，属于**全等级常驻基础掉落**（各自 0.05% / 0.015% / 0.05% / 0.1%）。
> 3. **概率为独立判定**：每个物品按自身概率（÷ 1,000,000）独立 roll，互不影响，一只怪可同时掉落多个世界物品。

### 9.4 等级块边界识别方法

由于物品 ID 也是数字，**定位等级块时不要按行号找**，要按 `-1` 分隔符 + 头部等级号定位：

```
[world drop]
    1   0          ← 等级块 1 起点
    ...
    -1             ← 等级块 1 结束
    2   0          ← 等级块 2 起点（每个 -1 之后紧跟的 <等级> 0 即新块开头）
    ...
    -1
    3   0
```

- 想修改 **N 级** 的世界掉落 → 找到头部为 `N 0` 的那个块；
- 每个 `-1` 后下一行的 `<等级> 0` 就是下一个块的起点。

### 9.5 修改实操

#### ✅ 需求 A：提高某物品在特定等级段的掉率
定位目标等级块，改对应物品行的第 2 列（基数 1,000,000）：
```
原：8669	500          ← 500 ÷ 1,000,000 = 0.05%
改：8669	100000       ← 100000 ÷ 1,000,000 = 10%（提高 200 倍）
```

#### ✅ 需求 B：在某等级段新增一个掉落物品
在目标等级块内、`-1` 之前任意位置插入一行：
```
    12345	100       ← 新增：物品 12345，10% 掉率
```
> 物品 ID 必须是 PVF 中真实存在的编号，否则掉落无效。

#### ✅ 需求 C：禁用某物品在某等级段掉落
把它的掉率改为 `0`（保留行），或直接删除该行。

#### ✅ 需求 D：让某物品只在高等级掉落
低等级块对应行保持 `0`，高等级块改为非 0。可直接参照 `1106 / 1107 / 1108` 的分段写法。

#### ✅ 需求 E：扩展等级上限（新增 201 级块）
在最后一个等级块（`200 0`）的 `-1` 之后、`[/world drop]` 之前，按格式追加：
```
    -1
    201	0
    <物品对>
    ...
    -1
```

### 9.6 注意事项

1. **`-1` 分隔符不可省略/错位**：等级块之间必须各有独立成行的 `-1`，缺失或多余会导致整个掉落表解析错位。
2. **头部第二列保持 0**：`<等级> 0` 中的 0 是保留位，勿改成其他值。
3. **等级号必须递增且连续**：从 1 开始，块顺序与等级号严格对应。
4. **物品对必须成对**：每行两个数字（ID + 掉率），出现奇数个数字会让后续全部错位。
5. **概率基数为 1,000,000**：`1000` 只是千分之一（0.1%），**并非必掉**；要 100% 必掉需填 `1,000,000`。数值可超过 1000，上限即 1,000,000。
6. **保持 Tab 分隔与缩进**风格，改完需**重新打包 PVF** 方能生效。

### 9.7 与怪物掉率文件的关系对照

| 维度 | `itemdropinfo_monster*.etc` | `worlddrop.etc` |
|------|----------------------------|-----------------|
| 触发主体 | 怪物死亡 | 独立于怪物，按等级段全局生效 |
| 控制内容 | 稀有度分布 + 物品等级范围 | 具体物品 ID + 各自掉率 |
| 分段方式 | 地图等级（`[item drop ref table]`） | 等级块（1~200，`-1` 分隔） |
| 概率单位 | 百万分比（累计阈值差） | 百万分比（每物品独立 ÷ 1,000,000） |
| 结构标识 | `[xxx]` 段落名 | `[world drop]` + `-1` 分隔的等级块 |

> 📌 **叠加关系**：怪物死亡时，**先按怪物掉率表决定稀有度与物品等级**，**再独立判定世界掉落池**中各物品是否掉落。两套系统并行生效，互不干扰。
