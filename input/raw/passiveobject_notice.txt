结论：
自定义的passive object无法指定生成方式，即po的最后一个数字，只能使用默认的0

在释放passive object时，写法类似与：
		99902	0	0	0	99	0 	``
最后那个字符串代表ptl。
前一个数字代表这个passive object的生成方式。 
0 代表正常生成
1 代表陨石降落
3 代表竖直降落。

但是这个标记对于自己自定义的passive object时没用的，会导致po无法生成。
原因是有的passive object是被写在exe文件里的，使用这个标记就会有相应的生成动画。

证据：
就是我们可以复制一个po，比如： 
48030
`Common/ShootingStarItem.obj`  流星落陨石

简单的把这个po在passiveobject.lst中复制一份，新给一个id，比如： 99901，
然后我们装备中引用新的id，： 
[passive object]
48030
0
0
0
99
1
""
[/passive object]
是只有地上的提示，没有实际的陨石的。

还有，如果我们把48030  `Common/ShootingStarItem.obj` 这个文件给复制一份，改为自定义的文件名，并且里面的ani act原封不动引用原来的，这种po，如果我们给的id是48030，即把原来的陨石给覆盖掉，我们的特效也就是po是会正常出来的。
但是一旦我们把这个po的id改为自己自定义的，比如99902，再去引用这个po又会出现只有提示光标没有陨石的情况。
